# 爆炒江湖厨神计算器 (bcjh)

自动计算《爆炒江湖》每周厨神大赛的最高分菜谱厨师组合。

**特点：纯 Python 实现，无需浏览器，内存占用 < 50MB，轮次控制确保跨平台结果一致。**


## 使用方法

当用户请求厨神计算时：

### 第一步：获取用户身份

使用 `/whoami` 获取当前用户的 User id（如 `03235020140026083370`），并在后续命令中通过 `--user-id` 传入，实现多用户缓存隔离。

### 第二步：启动计算

使用 `sessions_spawn` 启动后台任务：

```
使用 sessions_spawn 工具：
- task: "运行爆炒江湖厨神计算器，执行 scripts/bcjh_greedy_pure.py --user-id <用户ID>，完成后报告结果"
- mode: "run"
- cwd: "~/.openclaw/workspace/skills/bcjh"
```

如果用户提供了校验码，在 task 中说明：
```
运行 scripts/bcjh_greedy_pure.py --user-id <用户ID> --code 用户提供的校验码
```

常用参数：
- `--user-id 用户ID` — 用户身份标识，用于缓存隔离（由 `/whoami` 获取，自动传入）
- `--code 校验码` — 首次使用必填，之后按用户ID自动缓存
- `--force-import` — 强制重新导入校验码
- `--optimizer hybrid` — 混合优化（默认推荐，多轮SA+爬山）
- `--optimizer simulated_annealing` — 纯模拟退火优化
- `--optimizer local_search` — 贪心+局部搜索优化
- `--sa-reheats 24` — 模拟退火重加热次数（控制搜索深度，跨平台一致），默认24
- `--sa-temp 800` — 模拟退火初始温度，默认800

> **调试备用**：旧版浏览器方案 `scripts/bcjh_greedy.py` 仍然保留，支持 `--no-headless` 参数显示浏览器窗口。

## 执行流程

1. **触发即忘**：主 agent 使用 `sessions_spawn` 启动 subagent
2. **后台运行**：subagent 独立执行 Python 脚本（约 3-6 分钟）
3. **自动推送**：完成后结果自动 announce 回主 session

```
用户请求 → sessions_spawn → subagent 后台运行 → 完成自动推送结果
   │
   └─→ 主 agent 可继续其他对话，无需等待
```

## 校验码获取方法

游戏内：点击头像 → 设置 → 白菜菊花 → 复制校验码

## 算法说明

- **混合优化（默认）**：多轮独立 SA+爬山融合，轮次控制确保不同硬件结果一致
- **模拟退火**：贪心初始解 + 模拟退火全局优化，支持菜谱替换和厨师替换邻域操作，按重加热次数控制搜索深度
- **贪心策略**：先选菜谱再配厨师，纯 Python 内置计算引擎评分
- **每步对比**：
  - 选项A：新增厨师位 + 最优菜谱
  - 选项B：现有厨师空位 + 最优菜谱
  - 选择得分更高的方案
- **局部搜索**：当规则有材料数量限制时，自动启用两步替换优化
- **快速评估**：`setup_chefs` + `eval_recipes_fast` 机制，厨师设置一次后仅替换菜谱，~150x 加速
- **目标**：3位厨师 × 3道菜谱 = 9个格子，最大化总得分

## 数据存储

用户数据按用户ID隔离缓存在 `scripts/.bcjh_user_caches/<user_id>.json`：
- 已有厨师、菜谱、厨具、遗玉等配置
- 不同用户的缓存互不干扰，支持多用户并发
- 下次运行传入相同 `--user-id` 自动读取，无需重新导入校验码
- 兼容旧版：未传 `--user-id` 时仍使用 `scripts/.bcjh_user_cache.json`

## 注意事项

1. 计算器更新厨神菜品时间通常在周五下午或傍晚
2. 如遇计算器数据为空，请等待网页更新后再运行
3. 脚本运行时自动开启「只展示已有厨师菜谱」选项
