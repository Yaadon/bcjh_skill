# 爆炒江湖厨神计算器 (bcjh)

自动计算《爆炒江湖》每周厨神大赛的最高分菜谱厨师组合。

**特点：纯 Python 实现，无需浏览器，内存占用 < 50MB，多进程并行模拟退火，自动按 CPU 核心数并行。**


## 使用方法

当用户请求厨神计算时：

### 第一步：获取用户身份

使用 `/whoami` 获取当前用户的 User id（如 `03235020140026083370`），并在后续命令中通过 `--user-id` 传入，实现多用户缓存隔离。

### 第二步：启动计算

在终端中运行：

```bash
python3 scripts/bcjh_sa_parallel.py --user-id <用户ID>
```

如果用户提供了校验码，追加 `--code` 参数：
```bash
python3 scripts/bcjh_sa_parallel.py --user-id <用户ID> --code 用户提供的校验码
```

常用参数：
- `--user-id 用户ID` — 用户身份标识，用于缓存隔离（由 `/whoami` 获取，自动传入）
- `--code 校验码` — 首次使用必填，之后按用户ID自动缓存
- `--force-import` — 强制重新导入校验码
- `--workers 0` — 并行进程数（默认 0 = 自动按 CPU 核心数）
- `--sa-reheats 0` — SA 总重加热次数（默认 0 = 每进程 12 次）
- `--sa-temp 800` — 模拟退火初始温度，默认 800


## 执行流程

1. **启动脚本**：在终端运行 `bcjh_sa_parallel.py`（多进程并行，约 30 秒）
2. **等待完成**：脚本输出最优配置和得分
3. **报告结果**：将输出结果整理后回复用户

## 校验码获取方法

游戏内：点击头像 → 设置 → 白菜菊花 → 复制校验码

## 算法说明

### 优化流程：贪心 → 模拟退火（含轻量爬山）

1. **贪心初始解**：逐步填满 3 厨师 × 3 菜谱 = 9 格，每步选增益最大的厨师/菜谱
2. **模拟退火（SA）**：6 种邻域操作（换菜 50%、菜谱交换 25%、换厨师 3-10%、双菜替换 5%、远端替换 4-6%、厨师交换 1-2%），温度控制探索广度
3. **双层评估**：SA 内循环用 `eval_plan_quick`（~18 次 cal_score）高频评估，刷新最优时触发 `apply_plan` 精确验证
4. **轻量爬山**：精确最优刷新时自动触发，不换厨师，用 `eval_recipes_fast` 快速磨分
5. **重加热**：温度降至下限后从精确最优重新出发，默认 12 轮

### 多进程并行

- `bcjh_sa_parallel.py` 自动按 CPU 核心数启动多进程，每进程独立跑完整 SA，取全局最优
- 目标：3 位厨师 × 3 道菜谱 = 9 个格子，最大化总得分

## 数据存储

用户数据按用户ID隔离缓存在 `scripts/.bcjh_user_caches/<user_id>.json`：
- 已有厨师、菜谱、厨具、遗玉等配置
- 不同用户的缓存互不干扰，支持多用户并发
- 下次运行传入相同 `--user-id` 自动读取，无需重新导入校验码
- 兼容旧版：未传 `--user-id` 时仍使用 `scripts/.bcjh_user_cache.json`

## 注意事项

1. 计算器更新厨神菜品时间通常在周五下午或傍晚
2. 如遇计算器数据为空，请等待网页更新后再运行
3. 脚本运行时自动开启「只展示已有厨师菜谱」选项

- Python 3.10+（无第三方依赖，需 `@dataclass(slots=True)` 支持）
